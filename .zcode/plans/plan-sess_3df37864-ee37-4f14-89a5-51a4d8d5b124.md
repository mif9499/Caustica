# Bloom 最佳实践重构计划

## 设计思路

**当前方案问题：**
- 仅 3 级 mip（½→¼→⅛），有效半径只到 8~16 px，主流游戏是 64~128+ px
- 阈值在 2×2 box 平均后做，小光源（远处火把等）被稀释丢失
- 每级需要单独的 Gaussian H+V pass，昂贵且冗余
- 3×3 tent 上采样滤波太弱，方块感/闪烁

**目标方案：双滤波金字塔（Dual-filtering Bloom，COD/UE 风格）**
- 13-tap tent 降采样滤波器自带高斯质量的模糊，**不再需要显式 Gaussian pass**
- 阈值在**全分辨率**提取，保留所有微小光源
- 6 级 mip（½→¼→⅛→1/16→1/32→1/64），最大半径 ~83 px
- 9-tap tent 上采样，平滑无混叠

```
新流程:
  hdr → [全分辨率 threshold] → mip0 (13-tap tent ds) → mip1 → mip2 → mip3 → mip4 → mip5
  hdr ← [final composite] ← mip0 ← (us) ← mip1 ← (us) ← mip2 ← mip3 ← mip4 ← mip5 ←┘
```

## 文件变更清单

### Shader（4 文件）

**1. `shaders/display/bloom_threshold.comp` — 重写**
- 从半分辨率改为**全分辨率**操作（workgroup 覆盖 displayW×displayH）
- 绑定：hdrInput(全分辨率 in) + mip0(half-res out)
- 逻辑：对每个输出像素对应的 2×2 输入块，逐像素做 soft knee threshold（非平均后做），再用 13-tap tent 降采样
- Push constants: threshold(float), knee(float)

**2. `shaders/display/bloom_downsample.comp` — 重写**
- 用 **13-tap tent 滤波器**替换当前 4×4 binomial
- 13-tap diamond pattern（中心 + 4 轴 ±1 + 4 轴 ±2 + 4 对角 ±1）
- 归一化总权重，提供 ~σ=2.5 的模糊（在源分辨率等效）的同时降采样 2×

**3. `shaders/display/bloom_upsample.comp` — 重写**
- 用 **9-tap tent 滤波器**替换当前 3×3 tent
- 5×5 采样窗口 (radius=2)，tent 权重衰减
- 保持 additive composite：dst += upsample(src) * intensity
- 中间 upsample 传 intensity=1.0，最终 composite 传用户配置的 intensity

**4. `shaders/display/bloom_blur.comp` — 删除**
- 不再需要，13-tap tent 降采样已提供所需模糊

### Java（4 文件）

**5. `RtBloom.java` — 重写核心逻辑**
- `MIP_COUNT`: 3 → 6
- 移除 `ping` 字段（不再需要 Gaussian ping-pong buffer）
- 移除 `gaussian()` 方法
- `ensureResources()`：创建 6 个 mip image（½→1/64），移除 ping 创建
- `record()` 新流程：
  ```
  1. Threshold + ds: hdr → mip0（全分辨率阈值，一 pass）
  2. Downsample chain: mip0→mip1→mip2→mip3→mip4→mip5（5 个 downsample pass，由 radius 控制深度）
  3. Upsample chain: mip5→mip4→mip3→mip2→mip1→mip0（intermediate，intensity=1.0）
  4. Final composite: mip0 → hdr（× intensity）
  ```
- 新增 `radius` 控制：跳过超出 radius 范围的 mip 处理

**6. `RtBloomPipeline.java` — 精简**
- 移除 blur sub-pipeline（descriptor set layout/pool/set/layout/pipeline、dispatch 方法、destroy 中的 VK cleanup）
- 移除 `setBlurImages()`、`dispatchBlur()` 方法
- `dispatchThreshold()` 不再接收 displayW/displayH 且直接算 mip0 尺寸（保持兼容）
- `dispatchDownsample()` 和 `dispatchUpsample()` 不变

**7. `CausticaConfig.java` — 参数调整**

| Setting | 旧 | 新 | 说明 |
|---------|-----|-----|------|
| `SPREAD` | IntSetting, 0-3, default 3 | → `RADIUS`, 0-6, default 5 | 控制金字塔深度 |
| `INTENSITY` | FloatSetting, 0-3, default 0.2 | 不变 | 最终混合强度 |
| `THRESHOLD` | FloatSetting, 0-10, default 0.0 | 不变 | 亮度阈值 |
| `KNEE` | FloatSetting, 0-2, default 1.0 | 不变 | 软过渡宽度 |

系统属性和 TOML key 做对应更新：
```
caustica.rt.bloom.spread → caustica.rt.bloom.radius
bloom.spread → bloom.radius
```
`spread()` accessor → `radius()` accessor

**8. `RtVideoOptions.java` — UI 更新**
- `bloomSpread()` → `bloomRadius()`
- `IntRange(0, 3)` → `IntRange(0, 6)`
- 工具提示 key 改为 `caustica.options.rt.bloomRadius`
- 显示标签从 "Bloom Spread" → "Bloom Radius"
